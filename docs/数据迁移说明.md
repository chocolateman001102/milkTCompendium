# 奶茶图鉴 - TestFlight 版本数据迁移指南

## 问题说明

当你从本地开发版本切换到 TestFlight 版本时，**iOS 会完全卸载开发版本并删除所有数据**，包括：
- 所有饮品记录（SwiftData 数据库）
- 所有照片（Application Support 目录下的图片文件）
- 所有应用设置

这是因为虽然两个版本的 Bundle ID 相同（`com.choco.milkTCompendium`），但签名证书不同：
- 开发版本：使用你的开发证书
- TestFlight 版本：使用 App Store 分发证书

iOS 会将它们视为来自不同来源的应用，安装 TestFlight 版本时会先卸载开发版本。

## 解决方案

### 第一步：导出数据（在安装 TestFlight 之前）

1. 打开你的**本地开发版本** app
2. 在主界面右上角，点击 **⋯**（三点菜单）
3. 选择 **"导出备份"**
4. 等待导出完成（会显示"导出成功"提示）
5. 点击 **"分享"**，然后选择保存位置：
   - **推荐**：保存到"文件" app（iCloud Drive 或"我的 iPhone"）
   - 或者通过 AirDrop 发送到 Mac
   - 或者保存到其他云盘（百度网盘、微信等）

导出的文件名格式：`奶茶图鉴备份_2026-06-24.zip`

### 第二步：安装 TestFlight 版本

1. 确认已经导出并保存好备份文件
2. 打开 TestFlight app
3. 安装"奶茶图鉴" app
4. 安装过程中会自动卸载开发版本

### 第三步：导入数据到 TestFlight 版本

1. 打开新安装的 **TestFlight 版本** app
2. 在主界面右上角，点击 **⋯**（三点菜单）
3. 选择 **"导入备份"**
4. 在文件选择器中找到之前导出的 `.zip` 文件
5. 选择文件，等待导入完成
6. 看到"成功导入 X 条饮品记录"的提示

完成！你的所有数据和照片现在都在 TestFlight 版本中了。

## 备份文件说明

备份文件（`.zip` 格式）包含：
- `data.json`：所有饮品记录的数据
- `images/` 文件夹：所有原图和贴图

这个备份文件可以：
- ✅ 在不同设备间迁移数据
- ✅ 作为长期存档保存
- ✅ 重复导入（不会产生重复记录，会自动去重）

## 注意事项

1. **一定要先导出再安装 TestFlight 版本**，否则数据会丢失
2. 备份文件建议保存在云盘或多个位置，以防丢失
3. 导入时会保留所有评分、备注、日期等信息
4. 图片也会完整保留
5. 如果已经不小心安装了 TestFlight 版本并丢失了数据：
   - 如果有备份文件，直接导入即可
   - 如果没有备份文件，数据无法恢复（iOS 沙盒机制导致）

## 未来版本之间的更新

以后从 TestFlight 版本更新到正式 App Store 版本时：
- **不需要**手动导出/导入
- 数据会自动保留
- 因为它们使用相同的分发证书

只有从**开发版本**切换到**分发版本**（TestFlight 或 App Store）时才需要手动迁移。

## 技术说明

导出/导入功能特点：
- 使用标准 ZIP 格式压缩
- JSON 格式存储数据（方便查看和编辑）
- 自动处理图片文件
- 支持增量导入（不会覆盖现有数据）
- 兼容未来版本的数据格式
